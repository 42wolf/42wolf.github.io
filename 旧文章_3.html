github由https转换成SSH登陆仍然需要用户名密码的问题

2013/4/16

我刚才在设置我的github的时候发现，https的push实在是太麻烦了，于是按照官方的教程配置好了SSHkey，可是配置好了以后.

git push

还是会出现提示用户名和密码的问题。 不对啊！我可是明明已经配置好了啊，怎么还是会出现这个问题呢？ 于是我上网查了下，有人说应该用

git push git@github.com:user/repo.git

于是我试了一下，果然可以！ 可是如果我Push一下都这么麻烦，那我用SSH干嘛！不就为了少输入下用户名和密码么！ 于是我搜索了一下，我猜，应该是这两种PUSH方式决定的，SSH的Push和Https的Push应该用的是不同的链接，如果使用的不是自己的链接，应该就是需要用户名密码来确认身份的，而SSH配置好了以后，直接push push 不上的原因是，我在初始化仓库的时候，把默认的push链接变成https的了，然后我直接push 都是push Https的链接！ 我猜应该是

git remote add origin https://XXXX

这段代码把默认的push链接改成https了，于是我在自己尝试改了下。

git remote add origin git@github.com:user/repo.git

可是上面显示已经初始化过仓库了，囧~于是我Google了一下，终于在官网的Faq找到了解决办法

git remote set-url origin git@github.com:user/repo.git

哈哈，然后git push就搞定了，nice !enjoy!

莫名其妙

2013/3/12

在学校玩着游戏，舍友说要去南京找工作，于是我莫名其妙的收拾收拾东西，随便凑了个简历，也跑来南京了。

结果被一家做PHP的公司看到了，于是叫我去面试，我说除了C语言，我什么都不会，面试我的老大说，有C语言基础就好，php很好学的，于是莫名其妙的拿着不到做嵌入式一半的工资就去了，说不清楚为什么，稀里糊涂的就上班了。

找工作的时候，老舅说有个亲戚在马鞍山银行当行长去找找他看，老爹说你哥在合肥一个液晶生产公司里做主管，老师说，干嵌入式吧，嵌入式有前途。我一个都没有听，我就是选上了做拿钱最少，而且自己没有任何基础的互联网，因为我喜欢，仅此而已。

老师以前和我说过，人真的需要逼自己一把。而我理解的就是，人必须穿越痛苦的荆棘才能成长，现在呢，再等半年看吧，我能成长到什么样子呢？

我其实也很期待。

信息焦虑症

2013/1/25

信息焦虑

比如说SNS啊，过一会儿就想刷新一下，比如说微博啊，一看到技术牛人说几句有趣的新奇的东西就想顺着链接点下去，点点点，点到后面什么都没有学到，反而浪费了大量的时间。

我现在必须要改掉自己不够专注的坏毛病。因为我的专注力实在是太差了，一点非常不重要的小事都能让我从正在做的一件相对来说非常重要的事情里跑出去。学习的时候就这样浪费了很多的时间，我想我可能患上了信息焦虑症，所以为了防止自己再被这些东西打扰，我决定。

清理一下我的信息流。

以后主要上这几个网站：豆瓣、V2EX、Github，其他网站尽量少上。清理了我的GoogleReader，只订阅几个跟自己学习方向有关的博客。

多花时间去阅读书

坚持在书上能找到的就去看书，其次再到网上找，因为一上网太多乱七八糟的事情一下子就打乱我学习的节奏和状态了，而且很多知识在网上找到的都不具体和系统，主要还是解决问题的方法居多，对发现的问题的具体原理的分析往往很少，或者没有。

寻找自己需要专注的领域

李开复老师说：“follow your heart…”好吧，虽然李老师是个“鸡汤哥”，但总算对当年迷茫的我有着很大的帮助的，这个暂且不提。。

我需要在我专注的领域投注时间，才能达到我学习的最终目的：获取知识。因为自己以后的学习方向自己定下来是做web后端的工作，所以我需要在这些方面持续锻炼和学习倾注时间。

1、算法

2、linux

3、C/C++

4、Python/Ruby/Php

5、数据库

不要在不相关的东西上倾注太多的时间

最近老是纠结于工具啊，平台啊什么的，反而在实际的学习中却花费了很少的时间。

shit！决定以后找最简单的最容易入手的先学习，掌握基本的概念以后再尝试更换工具或者

平台，简单的说就是，快速试错，快速学习。

需要专注起来了

2013/1/20

今天安装vim的插件的时候，发现自己以前曾经配置过这样的VIm插件的，可是自己却完全忘了，和第一次配置的时候一样，谷歌与百度共上，键盘与大脑齐飞 ，后果就是嘴里吐出一句话[和谐社会要屏蔽]上次用了半个小时，这次用了40分钟。。。
发现自己很多事情都是这样，舍本逐末。
我刚学C语言的时候总是想着用linux来开发，用Gcc用Vim，然后各种命令行，想我学会了一定会很爽 ，巴拉巴拉的。于是不去学C语言，改去学Linux，改去学Vim，改去学Gcc…结果呢，花了大量的时间了解了以后，却又坚持不下去，不去写。慢慢的又忘了C语言该怎么写了。。于是我想，自学了三年多的计算机，为什么我依旧还是个菜鸟？因为不够专注和坚持不够。
老师建议我 “只做一件事情”，我觉得可以理解为在某个特定的时间，特定的只做一件事情，就是说：“要专注”，我不是说Linux啊，Vim啊什么的不应该学。而是，抓住事情的本质，花费最宝贵的时间学对自己最重要的东西。
由此可以举一反三，延伸到学习生活上，我发现，那些往往坚持做一件事情的人，往往就能得到他自己想要的东西 。万科公司当初为什么叫做万科，因为他当时什么都做，可是发展却很缓慢。后来，万科大刀阔斧的改革，改变策略，只做一门房地产，结果，大家都知道，它现在是中国最大的房地产商。
所以我现在决定自己以后的学习方向，就是做web后端程序员。我也愿意成为10000小时天才理论的践行者， 10年的时间，我会给自己和所有人一个惊喜。
我相信专注的力量。
操作系统需要我系统的学习，我最近需要学习并了解的语言有：PHP/Python/C/C++（重点练习)/HTML/CSS /JavaScript

因为毕业设计是做我们学院的网站，而且为了加深对这方面的理解，所以不想过度使用IDE，希望用vim+git来开发，所以，这两者就是我必须学习的了，而且Vim和Git都是非常有用的工具。

因为有过类似的教训，这次对于工具的学习。采取的策略就是赖世雄老师的话来说就是：“多就是少，少就是多”，决定每天花费一个小时的时间来通过有关的原版书或者英文文档来学习，不看中文文档，顺便提高下自己的E文水平。上机操作也是很有必要的。

每天都要敲代码，保持手感。

《超级快速阅读》读书笔记

2012/8/23

这本书我花了大概两个小时左右的时间来看，因为时间管理的东西和我以前在其他时间管理的书籍上看的都差不多，所以这部分我只是大致的浏览了一遍，主要看的是作者怎么讲快速阅读和快速记忆的。
当然，如果你是《30天学会XXX》，《7天搞定XXX》等书籍的爱好者，我想这本书并不适合你，这本书教你的是如何避免“浪费精力”和“高效学习”，如果你读了以后，并不对其加以实践，那么，读这本书的意义又在于什么呢?

这本书主要讲时间管理的部分
主要综合了三种时间管理方法：
1、建立个 to do list ，然后按照list 来实行，这个很重要 ，用我们的经验来说，当你有一件事情想去做，但却总是拖着不去做的话，你的潜意识里总会想这件事，读书的效率会大打折扣，往往会导致心情烦躁其实关于to do list ，还可以参考《getting things done》，中文译名《尽管去做》（这本书我才刚看一半，感觉不错）http://book.douban.com/subject/1085660/
2、每高效的学习30—45分钟，就开始休息5分钟，这其实不就是番茄学习法么……
http://www.douban.com/note/135148119/
3、每隔一段时间就开始复习，防止遗忘。。。。这，这不就是艾森豪斯记忆曲线理论么……
http://baike.baidu.com/view/342589.htm

作者这里唯一的与众不同的是，他很好的通俗易懂的用“缘脑”和“脑干”理论很好的解释了这个现象，有些意思。
这里时间管理的作用是在于，释放你的压力，让你更加轻松的完成既定的阅读任务。

然后就是我们最关注的快速阅读的方法
1、通读全书，理清脉络
拿到一本书的时候，不应该一下子就钻进去，而是应该首先通过大约五分钟的时间，通过浏览书籍的目录，每章的标题，重点快速的浏览一遍，给自己首先建立个大体的轮廓，了解这本书主要讲的是什么？作者观点是什么？论述顺序是什么？作者这里是利用思维导图，就是他所说的”视觉卡片“来建立学习的大体轮廓的，不过我觉得可行性似乎稍微差了点。
2、学会主动阅读
主动的总比被动的好，当然，某种程度上来说，阅读大部分都是主动行为，不可能别人按着你的头，使劲把你往书本上按，逼着你看嘛。这里主动的意思是，对自己阅读到底内容进行深入思考，以及给自己提出疑问，这样的效果是最好的，对阅读的理解是最深刻的。
3、如何训练
这里是这本书的核心的东西，具体实现有用手划读，跳读，提升阅读半径等等等等……就不一一阐述了
虽然看起来比较简单，但是实际操作起来还是有一定难度的
4、五感并用
学习一样东西，你所用到的感官器官越多，你对于理解明白这样东西的体会就越深刻

主要就是这么点东西。
这本书其实自己的东西并不是太多，感觉各种时间管理，思维导图的东西都有一点，不过作者将其结合的非常好，这种方面的学习方法可以尝试一下，但这书除了训练阅读速度的方法以外，并不适合文学，散文之类书籍的阅读，因为这种快速阅读方法主要目的是为了获取知识，而文学作品的目的和获取知识其实并不一样吧。

博客的定位

2012/7/3

之前建立博客半年的时间里，由于自己对博客的定位不清晰，和自己做事太过懒惰和喜爱敷衍的坏习惯，不经过大量的思考就写了一大堆的东西，从而导致博客文章的质量不高，也不符合我建立这个博客的初衷，觉得很不满，于是愤然删除掉了所有没有价值的文章，重新审视自己建立这个博客的原因

2012年2月7日，我在学校论坛站长番茄的帮助下，在一个国外的虚拟主机上搭建了这个博客
其实自己本来建立这个博客的目的在于希望在互联网上建立一个能够表达自己想法的地方，去发现并认识爱思考，爱学习的朋友，共同成长一起进步。
刚开始建立博客的时候副标题：
【成长—关注思维、学习方法、以及程序设计的东西】
写的不够宽泛，不符合自己写的东西的风格。
现在经过一番考虑，于是乎把博客的副标题改成了
【 成长——我们一直在经历】

因为成长就是不断的经历，不断地思考，不停地进步的过程。由于这个博客是我记录自己成长过程中的思考部分。思考是为了更好的完善，更好的经历。
既然一切都与经历有关，所以【一直在经历】这个副标题，就是我想要的。
2012年8月9日19:46:30修改
之前自己也是在自欺欺人【一直在经历】这个副标题，究竟是经历了什么？
无非是自己没有东西可写，文章散漫没有主题而给自己的一个借口罢了
现在终于想到了自己博客发展的方向
【阅读，思考，行动，成长】
因为自己比较喜爱阅读，希望自己能够通过不断的阅读和思考，然后将思考付诸于行动，达到自己个人成长的目的。

拖延症和焦虑

2012/3/31

最近在努力克服自己的拖延症和焦虑感。

发现自己一直以来都依旧很难集中精力干一件事。例如背单词时候经常会冒出系统要不要重做，清明放假回家以后到底吃什么的奇怪的乱七八糟的想法。这种学习习惯的确是很可怕的，因为人的精力是有限的，就拿计算机来说吧，计算机操作系统是可以多任务同时运行的，譬如你可以在用电脑听音乐的时候一边上网一边和家里人视频，这就是传说中的：“多任务运行。”

但是大家经常在拷贝文件到U盘的时候，有没有这样的经验，同时复制两个文件加起来花的时间还不如先拷贝好一个文件，才去拷贝另外一个文件加起来的时间多。为什么会这样呢？因为操作系统在执行多任务的时候实际上是时刻不停的在各个进程里切换的，这些进程虽然看起来是同时发生的，其实不然，他们是在以很快的速度不停地切换，切换，切换，人是感觉不出来的。虽然这样某些时刻比较方便，但是它在切换的过程中却损失了大量的时间。人脑也是如此，只有在大脑精神高度集中的做一件事情的时候，人的工作效率才能达到最高值。曾经我也和老师交流过关于学习的东西太多很难掌握的问题，老师给的建议就是——只做好一件事情。

在我看来很多看起来很聪明的人学习效率很高的同学比起我们最大的不同就是做事情的专注和认真。

最近锻炼自己，每每到学习不够专注的时候，就提醒自己，集中注意力做好一件事，才是成功的不二法门。
王小波说：“人最本质的痛苦大概就是自己的理想和自己现实的无能之间的差距”。我想我的大部分焦虑 可能便源自于此，看了刘未鹏在博客上推荐的书，然后找了卡耐基的《人性的弱点》来看，果然是本好书，我利用走路吃饭上厕所的闲暇的时间把这本书给看完了，焦虑感果然有了明显的改善。
不过某种拖延症的习惯真的很难去改变，就好比今天下午面对厚厚的C语言书没有去看，产生了畏惧感，于是跑回寝室睡了一下午一样。。

刚刚有所改变，立马就是各种意志不坚定，然后就是各种悲剧……TAT

今天在寻找好的治疗拖延症方法的时候，突然看到一个观点，就是说一个人不停的寻找治疗拖延症方法的过程其实也是拖延症的一种表现。。

难道我已经陷入其中了么？

古人说知易行难，事实证明的确如此，知行合一是最难做到的。

坚持去做就好了吧，一切都会好起来的。。。
