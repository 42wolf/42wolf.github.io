浅谈扎金花技巧

2014/2/16

过年的时候回家玩，在我们这偏僻的小地方，唯一的娱乐就是赌钱了，于是我过年回家就开始了长达一个礼拜的赌博佬的生活。

刚开始的几天我总是输钱，这让我很苦恼啊很苦恼啊有没有！！！

对于我这个坚持理性世界观和方法论的人来说，这种遭遇完全是不符合我对这个世界的认知的，因为按照概率论来说，我拿到好牌的概率应该是和别人一样的，我百思不得其解。于是这几天输了很多钱，没错，是一大把钱！！！

同学们解释为我运气不好，对于我来说，这个论断是没有任何实际根据的。因为我从来不认同“运气”和“命运”这种事情，我觉得任何事情都是有理由的，没有理由和原因的事情是根本不存在的。

然后我回去看《博弈论》，说到纳什均衡，纳什均衡指的是在非合作博弈里面，不管别人怎么选，自己都可以在某种程度上达到最优。然后我恍然大悟，我输钱的原因不在于概率的问题，而是在于我花在每个概率上加权的砝码不一样，也就是说局部非最优的场次特别多，导致全局最优的概率变小，输钱也在情理之中了。

所以加权的赢钱方式是：（收入-投入）对手拿到比你牌小的概率 最后加权赌钱的局数
而投入的合理公式应该是：投入= （收入/投入）你判断对手牌比你小的正确率*常数系数
收入的公式应该是：收入=自己的投入（心智，财富状况、心情、习惯）+他人的投入（心智、财富状况、心情、习惯）

在这里面，起决定作用的是，

1、你的心智
2、别人的心智
3、对手拿到牌比你小的概率
4、你判断对手牌比你小的概率

心智在这里怎么理解呢,可以理解为。控制自己，不要情绪化出牌，不要有所谓的不信邪的心理，不要在输的时候情绪化使劲上钱赌博。

该怎么做的时候就该怎么做，比如单A可以上的时候，不要因为输了钱就不上，跑了，这样只会导致越输越多。也不要因为输了钱就使劲想捞回本，

小点就刷涮的往上上，这也是错误的心态。坚持自己赌钱的原则，肯定就会赢。

然后就是，对手拿到牌比你小的概率，这个是没有办法控制的事情，当然，这个我们是可以依靠自己手里的牌计算出别人拿到比自己大的牌的概率的。

抓到金花的概率是多少，豹子的概率是多少，是可以计算出来的。具体的可以研究下概率论，就不深入讨论了。

因为和同学打着玩，所以也没必要这么认真的研究概率啦，挺累的。

看的头晕的可以记住以下几点简单的道理：

1、及时止损 比如说，人少的时候输的多就一开始就看牌。人多的时候就闷牌，因为人多的话，看牌早是会很吃亏的，闷牌的话投入产出比也很难说，如果经过专业训练，这个概率是可以计算出来的，因为人越少赚钱的概率就越大
2、越是那种情绪化严重的人，越会输钱。我观察过很多人，那种情绪化很严重的，控制情绪能力不强的人，一般就会输很多钱。我之前输钱就是情绪控制力上出了点问题和对概率论的理解偏差。

3、察言观色很重要，了解每个人的性格，就会增加你判断他牌大小的准确性。所以最好的赌钱表情是面瘫。当然，如果这个你学不会的话，也无所谓。及时止损和理性才是赌钱当中最关键的一点。还有，不要贪婪。
明白了这个道理，我输的钱都赢回来了。

其它具体的细节可以再自己研究嘛，真正的赌神，都是靠概率来赢钱的，不是靠运气和作弊。

confluence数据库校对规则的问题

2014/1/15

字符集和校对规则

字符集是一套字符和对应编码的集合，而对应的校对规则是对字符之间进行比较的一个或多个规则的集合。

举个简单的例子：

A——0

B——1

C——2

a——3

b——4

c——5

类似于上面的这样的，左边的集合属于字符集，右边的集合属于编码集。

而校对规则就是用于字符之间互相比较的一套规则：

比如我设定一个大小写敏感的规则，那么它就会把‘A’字符和‘a’字符当做两种不同的字符。

但如果我设定一个大小写不敏感的字符规则，那么它就会把‘A’字符和‘a’字符当做完全一样的字符来进行校对处理。

Mysql的校对规则和字符集设定

Mysql 5.1支持如下特性

使用多种字符集来存储字符串
使用多种校对规则来比较字符串
在同一台服务器、同一个数据库或甚至在同一个表中使用不同字符集或校对规则来混合字符串
允许定义任何级别的字符集和校对规则
由此我们知道，Mysql服务器、数据库、表单、列，这些的校对规则和字符集都是可以不一样的。

也就是说，你可以设定数据库的字符为utf8,校对规则utf8bin，表单的字符为utf8,校对规则为utf8general_ci，列再使用别的字符和校对规则，这是没问题的。

Confluence测试环境遇到的问题

上周在测试环境测试Confluence，我把我们线上环境里面的数据库导入到按照官网设置的空白数据库。

数据库的创建：

CREATE DATABASE confluence CHARACTER SET utf8 COLLATE utf8_bin;
数据库的导入

mysql – u root -p wiki < mysqldump.sql
启动confluence，无法启动，总是报编码校对的错误。显示报错 ‘Detected tables with non-default character encoding/collation’ Message

我按照官网提供的解决方法，修改了表单的字符校对规则，将所有80个表单的校对规则改成了utf8_bin.

类似于这样：

ALTER TABLE wiki.AO_187CCC_SIDEBAR_LINK CONVERT TO CHARACTER SET utf8_bin;
重新启动，结果发现还是同样的问题，编码校对的错误

然后经过同事的提示，去线上环境看了下，发现校对规则全部都是utf8generalci，于是改成了下面的格式

ALTER TABLE wiki.AO_187CCC_SIDEBAR_LINK CONVERT TO CHARACTER SET utf8generalci;
结果改完了还是这样的问题，最后彪哥解决了问题，重新创建了数据库，这样就可以了。

CREATE DATABASE wiki DEFAULT CHARACTER SET ‘utf8’;
原因就是，创建utf8字符集的数据库的时候，默认的校对规则是utf8generalci，我们的数据库最早创建的时候没有参考官方的这个教程，导致我们所有的表单也好，列也好，都是utf8generalci的校对规则。

然后我在写sql进行转换的时候，只转换了数据库和表的校对规则，更改列的校对规则的时候，报错了，当时也不知道是为什么。

总结一下：

解决这个问题的方法

1、改变新建数据库的默认编码（我们的办法）

2、转换整个数据库里面的数据的编码（官网的建议—但修改校对规则会有报错，当时没看懂，有空再验证一下）

念念不忘，必有回响—2013年总结

2014/1/5

1、读书

收获：

2013年读了68本书，平均五天一本：http://www.yuedudna.com/users/48599262/book/2013

视野得到了进一步的提升，见到的东西多了，看到的观点多了，自然思考就会有源头，不至于禁锢在一个狭小的范围内，所以我的收获就是

看到了更多的东西。

不足：

过于偏向于数量，却没有在阅读质量的提升上花费太多的时间。阅读笔记写的很少，总结也不够多。

新一年的展望：

希望能够建立自己的知识体系，以后阅读专注的方向：主要偏向于心理学，逻辑思维方面的书，计算机方面的数据库，Linux，需要深入一些。

算法和编程方面还是比较薄弱，需要进一步的加强和不断的练习。

2、交流

收获：

和组内的交流给了我很多书本上学不到的东西，譬如如何推动一件事情，如何坚持一些事情，如何去学习。

不足：

面对事情经常会有畏难情绪，对于比较dirty的活还是有些排斥，情绪控制的不太好，需要进一步的锤炼自己。和人交流的主动性仍然不够，只是在deadline的时候才会主动去寻求帮助。

需要改变。

新一年的展望：

控制自己的情绪，处理好杂乱的事情的同时也要多主动找人交流沟通，发现自己的不足。

3、思考

收获：

之前我每一天晚上都会花上一个小时的时间来思考自己遇到的问题，视野的提升给我的好处就是，思考问题稍微比较客观和从容了一些，愤青的时候更少了点。

不足：

发现虽然思考的不少，但是有时候情绪上来的时候容易钻牛角尖，听不去别人的建议，很多问题思考的太过理想化，以后要注意做事之前得多角度的思考不能太过偏激。

新一年的展望：

心态放平和一些，不能太过焦虑和急躁，否则容易走错路，做错事情。多听听别人的想法和建议，但是，坚持自己的原则，依然很重要。

4、成长

收获：

刚来的时候美团月销售额10亿，现在已经突破20亿了，时间却只有半年，但回头想想，很多事情都是在发生巨大变化的，不管是从团队，还是个人，还是我们在做的事情。

我在这大半年里，除了技能上的提高以外，思维方式和处事方式也有了很大的改变，当然，对于我自己来说，最大的莫过于自己视野的提升吧。

不足：

成长最慢的时期，恰好是自己情绪波动最大的时期，这个告诉我的一件事就是：“想的太多未必是一件好事。”当然，也不可能是坏事。缺点就是在能力不够的情况下，想法比较多，较浮躁。

新一年的展望：

之前上学的时候老师告诉我的一句话就是：“一个人有太多想法，和一个人没有想法，是没有区别的。”虽然我不太认同，但某种程度上还是很有道理的，多思考，谨慎下结论。

努力坚持，不是有一句话说嘛：“但行好事，莫问前程”。

一些感慨：

环境能造就人，是的没错。但我们作为一个人，我们和动物的差别就在于人能改变环境，就像我以前总是抱怨学校不好，工作刚开始的时候抱怨事情做的没挑战性一样，是弱者的表现。

做事情就像我读书买书看书，虽然很多人都不以为然，但我一直就这样做下去了，然后就潜移默化的会影响到很多身边的人开始读书，开始思考问题，我就觉得这是我的胜利。

有些时候我们为什么喜欢逃离，因为我们害怕。很喜欢《一代宗师》里面的一句话——“念念不忘，必有回响”

需要感谢的人有很多，就不一一说出来了。我不是王总的粉丝，但我挺喜欢他说的话：“对未来越有信心，对现在就越有耐心。”

用来与诸位共勉，顺祝新年快乐。

Mysql入门好书——《Mysql 必知必会》

2013/12/10

今年BootCamp的时候，我司的DBA给我们推荐了三本书：MySQL必知必会
高性能MySQL(第3版) MySQL技术内幕(第4版)，还有Mysql的官网。大神说：“读完这些东西，再加些丰富的经验，就可以去做DBA了。”

那时候还对数据库没什么太多的概念，因为根本没有做过相关的东西，有种“不明觉厉”的感觉，于是去Amazon上去买了一本《必知必会》，可看了几页根本没明白是什么东西，看一会儿就睡着了，于是就扔到了一边，跑去看英文文档了。

然后最近一段时间开始了一些线上的操作，总是在Google上找到一些现成的code 粘贴进去，强忍着看了一些英文文档，很多时候也不知道是为什么，反正活儿都干掉了。

今天又翻开这本书，突然觉得之前完全看不懂的地方几乎没有任何困难就看懂了，然后登录自己的VPS，比照着自己的博客里面的wordpress数据库给操作了一遍，不知不觉就搞定了几章，发现很多东西自己都是做过类似的工作的，之前却不知道意思，有种走了弯路突然豁然开朗见桃花源的感觉。

突然想起来我们之前一个经验比较丰富的同事告诉我，他学习的方法就是一件事情先找到方法给干掉，先知道怎么用，再系统的看书学习，借此和大家分享。

这本书讲的非常全，从基本概念，到查询到插入新建表，用户的管理，都有具体的例子，非常适合没有任何基础的同学来学习Mysql，看完以后可以看看mysql的官网

总之这本书学习的方法就是：1、十分钟了解下数据库的基本概念 2、找到练手的数据库 3、对照着上面的内容去敲。

最后就是，要多用。

半年关于工作的感受

2013/11/8

来美团也将近半年了，团队不太稳定，自己事情做的不太好，成长也比较缓慢，一系列的因素，都让自己做的很不爽，然后每天都很纠结的过，老是想着要不要离职，换一个公司试试，又不太舍得公司的环境，然后就老是处于一种很焦虑的状态，事情做不好，总是在周报上吐槽，吃饭的时候吐槽，微博上吐槽啊，说来说去觉得很累，自己也感到没什么意思。加上江哥也要离职了，更觉得再这样待下去真的很没有意思了。

然后彪哥和江哥找我，跟我谈了一大堆自己的问题，给我分析了一下我的问题，我突然醒悟到，自己原来真的还有很多很多不足。虽然之前自己也或多或少的意识到了，但人都是这样的，总是喜欢欺骗自己，每个人都需要把血淋淋的事实摆在眼前，才能正视自己的缺点。

江哥给我分析了一下我的缺点

基础技能差
做事浮躁，太过个性，容易情绪化，不喜欢做DirtyWork，但更高层次的事情却做不了
做事虎头蛇尾，很多事情做了一半就停掉了，没有很好的结果。
喜爱逃避问题，遇到点困难就不想做了。
做事情主动性不强，总是推动不了事情，需要别人Push我
交流能力差
回去想了想，不禁觉得自己丑陋不堪，一个人自己都已经如此愚蠢了，怎么会好意思有颜面来吐槽公司和别人呢？其实我们对别人和环境的不满和愤怒，很多时候都是因为对无能的痛苦和愤怒。

别人的问题

我以前犯过一个很愚蠢的错误，就是，喜欢从自己的角度去评论和判断别人，动辄因为别人的一些言论就骂人傻逼。在学校的时候，也经常看不起周围的同学，觉得别人是傻逼，大多数人，不知道他们怎么想的，或许也认为我是个傻逼吧。

事实上，我到最近才领悟过来，别人的成长环境是和你不一样的，别人的知识水平是和你不一样的，别人的一切一切是和你不一样的。

知乎上有一篇问答 能否说家境越好的人越愿意努力向上，而家境一般的人反而会安于平凡？

里面排名第一的回答引用银魂里的一句话说到：“和你们这些少爷不同，我们光是活着就竭尽全力了。”

这句话让我心有戚戚焉。

我就觉得我算是个比上不足，比下有余的人了。我想起了几个和我读一个初中的表哥们，他们和我一起背着书包上学，初中以后出门打工，而我继续读书，上高中的时候，他们还在打工，我上大学的时候，他们还在打工，我现在坐在办公室敲代码，他们还在打工。 我们生活的轨迹完全就不一样了，我技术再烂，再怎么说我也是个白领了，成天坐在办公室里，他们呢，现在是处于卖劳力的人啊，没有一技之长，成天被人呼来喝去，就为挣那点微博的工资。。。。

我不觉得自己多聪明和多努力，你瞧，我不就读了个很怂的二本么。我觉得我有今天还是因为我爸是老师，我家稍微比他们家环境好些，我不需要干活，我有人教，我有书读，后来我学会了上网，我有了环境。

可是我后来学计算机又觉得很不公平啊，为什么别人在上小学的时候家里就有电脑可以玩编程？为什么别人从小就能见识到各种各样大的场面？为什么别人都穿的很光鲜而我的父母节俭到成天穿着破着的袜子，连个好点的袜子都舍不得买？

现在我想明白这个道理了，很简单，因为别人跟你不一样。既然改变不了事实，那我们就不如鸡汤一把，听开复哥的，接受自己所不能改变的，改变自己能改变的吧，比如心态。

后来江哥跟我谈的时候，告诉我。我们这些做技术出生的，都有一些傲气，可能会看不起技术比我们差的小白们，看不惯能力不强却当上领导的人，看不惯很多在公司里欺上瞒下，溜须拍马的人。但是，你得明白，每个人都是要生活的啊，每个人都有老婆孩子，每个人都是有家要养的啊。你以为他们想要那样么？

是啊，为什么我只能看见别人的缺点和不足，却看不到别人背后的痛苦呢？我们出卖的是技术，而那些溜须拍马的人，出卖的可是尊严啊。想想，谁的代价更大呢？

自己的问题

说说我自己的问题

做事不主动
总是被需求牵着鼻子走，不知道什么是重要的？什么是我该做的，不该做的？我做了一件事情究竟有没有产出？

结果就是，我不知道该怎么下手，烦躁，不安，焦虑，恐惧。都会一股脑的袭来，把我的心智给控制住了，让我不知所措，诚惶诚恐。

事实上，好的做事方法是，自己把握好什么事情是最紧急的，最重要的，我能做的，然后依次解决掉他们。

关于说话
所以你把许多你自己觉得非常自豪，非常有用的东西跟别人说出来，别人可能会觉得没那么重要，或者别人根本就不关心。当然也不排除很多自己炫耀啊，情绪化啊的东西在里面，可是这一丁点的东西会让人觉得你这个人有问题，影响别人对你的判断。

就像王垠一样，很多人觉得他在秀优越感，其实我觉得不是的。他很多时候是在说实话，他是在分享他的感受，大多数人不喜欢他最根本的原因是因为，自己不如他，自尊受到了伤害，就会去骂他。王小波说：人最本质的痛苦，都是因为对自己无能的愤怒，我就特别喜欢这句话。

分享这种事情，一种是有用，一种是没用。

比如我告诉你，比如学习这个东西是没有捷径的，你大致应该怎么学，就行了。这个是有用的，但大多数人都不太喜欢接受事实和真相。反而喜欢另一种，李开复，俞敏洪之类的的什么改变和你所能改变的，接受你所不能改变之类的大家都知道的废话，然后告诉你，听我的，就能成功，让你打满鸡血，结果就是，你还是不知道该干嘛。

就像我曾经跟一个妹子大谈特谈手机刷linux，妹子觉得很没有意思，然后跟我说：你呆在自己的世界里太久了。以后我就明白了，于是我再也不跟任何人说自己专业和工作的事情，问起来就说是修电脑的。别人问这种问题大多都是为了起一个话头，或者只是为了礼貌？知道你大概赚多少钱？敷衍一下就好了，就像七姑妈八大婆跟问你对象和工资的事情一样，无非就是找个话头和谈资罢了。

还得需要时刻体会别人的感受，我这一点就做的很差，可能是我从小都只和性格豁达开朗的人玩的缘故吧。。

就像我老是吐槽前台的妹子不对我笑，这个问题其实很有意思，可为啥看见舒伟就乐开了花？长得比我帅？然后舒伟告诉我说，你丫整天给人板着个脸，别人还对你笑，别人该多贱啊。。

想想还是自己的问题，自己读书这么多年了，也认识很多人，但就是真正说的上是朋友的很少很少呢。而且有种越来越少的感觉，还是自己很少跟人说自己想的东西，说自己的感受，为别人着想，鼓励别人之类的。总想着怎么有意思和有技术含量的骂人，想来真的是很蠢啊……

公司的架构师夏华夏分享的关于钱的问题就很有意思，他说，一个人有多少钱是看这人有没有那个能力留住那么多钱的。你想想那么多中了百万大奖的人，过了不久就倾家荡产比中奖前还惨，就是因为他的能力不够，他承载不了那么多的钱，有再多的钱他也能给造掉……

突然想起来老大在我才来的时候给我说的任正非的例子，不过我现在想的有点不一样，就是，人想要说话，想要涨工资，想要做更有挑战性的工作，想要别人听你的，你就得先证明自己。

觉得工作不爽的时候，觉得工资低的时候，觉得没有成就感的时候，多问自己几句。思考好了再说话，一个人喋喋不休总是会让人不舒服，就像我妈跟我说话我都是左耳朵进右耳朵出，我爸说话我就能记住，因为我爸不轻易说话。 我是谁？凭什么？我该怎么做？我想做什么？

说话这真是个艺术，我觉得对于我现在来说，不知道什么样的话有什么样的后果，可能还是少说话比较好。

至少，可以表现的自己很踏实，说的越多，越能表现出自己的愚蠢。虽然我觉得我的愚蠢与我说多少话无关。
