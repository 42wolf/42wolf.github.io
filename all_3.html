我们究竟需要什么样的PM

2014/11/11

￼
我们究竟需要什么样的PM

前一阵子和其他的同学一起聊天，说到我们校招产品淘汰率之高的时候，有同学说我们是要求太高，然后我回去仔细思考了一下，究竟是我们team的要求太高，还是产品的本身就需要那么优秀的人呢？思考了一段时间，想了一下，我觉得我们不降低标准的思路是正确的。

于是乎，我便简单列举了一下我这段时间思考的产品需要的一些特质与技能，其中的大部分我自己都没有达到，籍此与大家一起来分享，至于为什么需要这些特质，以及如何去做，我也有了一点未经验证的想法，下一篇文章会与大家一起分享我的思考。

非专业要求

对于产品这个职位，我们更关注应聘者自身的素质远远大于他掌握技能的要求，对于个人素质我们的要求大致是这样的。

1、逻辑性强，聪明

2、沟通能力强，反应快，能够很快速的理解别人说什么,别人想要什么

3、执行力强，没有做完的事情都会立刻去push、去跟进

4、知识面非常广，各种领域的知识丰富，且有一定的深度

5、批判性思维，对事物有自己独特的见解，不人云亦云

6、同理心，即善于从别人的角度来思考问题

7、对事情充满好奇心，思维模式比较special，有探索精神，有激情

8、具有自我反省和自我修正的能力

9、做事有规划，能够坚持

10、面对问题和压力的时候很冷静，能够的给出特定情境，特定问题范围内最靠谱的解决方案。

专业能力的要求

专业能力方面，个人觉得，只要用心，肯花时间，基本上不会差很多，当然，应届生这方面几乎是没有要求的……如下这些都是我接下来想要学的

1、掌握市场调研的工具和方法（有一套成熟的理论）

2、数据分析方法，工具的使用

3、交互设计工具的使用，用户界面的设计

4、项目管理的方法，掌控项目的能力，相关工具的使用

治疗拖延与洗洗睡吧

2014/11/9

拖延症的专家

我一直以来，都是一个拖延狂。具体描述一点就是，如果想做的一件事情如果可以明天去干，我绝对不会今天去干，除了吃饭拉屎撒尿上班，啊，对，还有看毛片。

这件事情影响到我生活的各个维度，比如说我学了很长时间的编程到现在居然还只会捣鼓点Linux，编程能力几乎等同于没有。

后来这些问题愈发严重，以至于我上大学的时候很长时间内因此患上了一种叫做轻微抑郁症的东西，那时候我的脑子里成天想的问题是如何去跳我们学校的主楼，以及如何去跳会比较优雅，后来经过自我控制和阅读心理学的书籍，总算了解到了自己一些毛病产生的原因，抑郁的症状就自然好了。

然后也想靠这些方法去改掉自己爱拖延事情的毛病，从《getting things done》,《晨间日记的奇迹》再到《高效能人士的七个习惯》，一个明显的事情就是，我读过的战胜拖延症书籍超过了我见过的所有人，我也分析过自己产生拖延的原因，原因无非也就是那么几种，有时间的同学可以看看李笑来（他说他不喜欢别人叫他老师）写的书《把时间当做朋友》，但是我还是改不了，一切依然还是老样子……

因为这个问题一直在困扰着我，所以呢，我就经常会观察一下会有什么好的方法，我可谓是口上战拖的专家了，有时间的同学，可以找我，我给你普及一下拖延症的知识，呵呵呵……

没有黑魔法

美团外卖的老大王慧文曾经有个理论叫：『神秘主义依赖症』，怎么说呢，就是他创业这么多年，在遇到某些关键节点，不知道该怎么去做的时候，小弟们总会觉得大佬们有解决问题的方法，这就叫神秘主义依赖症……（其实大佬也是普通人，很多自己也是一头雾水……）用二次元的话来说，就是：这个世界没有黑魔法。

一个季度前，我还是比较蠢的（虽然现在也是），花了二百大洋去听了一下前搜狗输入法的产品经理马占凯老师的治疗拖延症的课程……

去的时候很高兴，回来的略微有些失落，然后仔细想想，突然意识到这些理论我全部都知道，而且因为阅读大量心理学书籍的缘故，我不但知其然，我还知其所以然，甚至毫不自夸的说，谈起这方面我比他更专业，因为这些是他作为二道贩子贩卖过来的知识，不专业的地方体现在，比如只给五分钟的时间让人写计划，五分钟的时间让人决定优先级，这些对于拖延症患者来说，这些都是很难做到的，即使很快写出来，也是不靠谱的行为……更不靠谱的事在于，马老师自己做到完全克服拖延症了么？除了健身这件事以外，当年弄的风风火火的章鱼计划呢？

当然，我这里想表达的意思并不是要抨击马老师的分享讲座，只是想说明一个事实。也是很多人在《后会无期》里面说的一句话，『懂得很多道理，却依然过不好这一生』，不停的去找彻底解决问题的办法，其实，就是不敢去面对现实而已。

后来我在team里面分享我去听课的笔记以后，老大告诉我：对于拖延症这种事情呢，方法论只能帮助你从0.1到1的量变，但是要想从0变到0.1，得看你有没有足够大的驱动力，就好像你在到了吃饭的点一般总不会不去吃，厕所该上的时候也一定会去上一样。如果一件事情你能够做到像吃饭上厕所那样迫切，目标明确，那么世界上就根本不会存在拖延症这种事情。说到驱动力呢，我的理解就像是老大说的，关键是看你得想清楚，你究竟希望过什么样子的生活，你想成为什么样的人。

我现在吸取了一些小的教训，便试着去先从一件事情做好，慢慢来做，希望可以下个季度的时候可以有机会能与诸位分享我战胜心魔的心得吧。

你永远都叫不醒一个装睡的人

2014/10/15

前几天刷朋友圈，有朋友说了如下一句话『忽然发现csdn弱爆了，ChinaUnix才是王道』，然后我告诉他，读点英文文档会比较好，然后我跟他掰扯了半天，他还是表示读中文文档比较好，读英文文档是装逼……然后我就突然想到了一些关于自己的事情

大概是四年前，那时候我在某师范学院读大二，因为无聊，就一个人坐在宿舍打开爸妈给我买的Y460，装上vc++6.0刷oj，成天叫着嚷着要做师院第一ACM大牛，当时哪怕坚持刷半年，那水平也是相当的不错了，可惜后来我只刷了三天，于是总会想起那天晚上的画面，月亮有点弯，天有点冷，在得知我是我们学校唯一知道ACM这个东西的学生之后，我有点恍惚的用一指禅敲着键盘，猴哥点了根烟，轻蔑的笑了笑，看着我说了一声：“来一局？”，我一恍惚，便掉进了逃避的窟窿，过了很久才挣扎着爬了出来。

毕业以后我来到美团上班，做了一段时间的运维做的很不爽，吃饭的时候就跟江哥吐槽，说自己想去学开发，然后江哥问了我一句，既然你想做，为什么自己不去学会了呢，说到底，你还是不想做嘛，然后我觉得有道理，然后又想了一年多…

所以后来看很多人的时候，我就会拿他来和我过去的状态做个比较，大概估计的话，我觉得应该不会差的太远。

自从有妹子说我太过自我以后，我便努力改变了很多，比如说少说点自己做不到的事情，比如说正确的认识到自己的不足，比如说少自我膨胀，等等…

但要一下彻底改掉自己几十年养下来的坏习惯，真的很难，只能一点点的去变，有同学大致跟我说过这样一句话：“知道自己的缺点却不去改正，是一件很搞笑的事情。”我就觉得他说的很对，我听了很惭愧，作为一个人应该起码是要有点自尊心的，最起码不能被自己鄙视，我现在就做不到，于是就感觉很不好…

虽然自己是个半吊子，却还总想着去帮别人，但最后结果总是不太好，于是便悟出一个人生哲理，一个吃屎的狗，永远不会教会另外一条狗不吃屎，但是它是有机会可以成为一个不吃屎的狗的。

我后来上大三，整天翘课去实验室玩，学着画点电路板，写点程序什么的，本来没什么，后来拿了一些现在看来毫无意义的奖，就有学弟找上门来要我带他，我给他拷了资料，给他指了个位子，就说先坐这儿吧，可以自己学，然后过学了几天，人就不见了，问起来总是说哎呀这个很难，那个不会，又没人教，当时我很想说去你妈的，然后我忍住了，挂了电话。

昨天又听刘昌毅说：“作为一个PM就是要证明自己在某种情境下对一件事情理解的比别人透彻。如果要说一个放之四海皆准的原则的话，那就是和菜头说的一句名言了，叫做：我说的都是错的。”因为把情景放大以后，道理就都是错的了。我觉得很多事情虽然现在我想得不对，随着时间的变化，总会想对的，至少还有希望。

但有句话叫做，你永远也叫不醒一个装睡的人，对于那种装睡的人呢，我们能做的就是，不要在不必要的人身上花费不必要的时间…

浅谈扎金花技巧

2014/2/16

过年的时候回家玩，在我们这偏僻的小地方，唯一的娱乐就是赌钱了，于是我过年回家就开始了长达一个礼拜的赌博佬的生活。

刚开始的几天我总是输钱，这让我很苦恼啊很苦恼啊有没有！！！

对于我这个坚持理性世界观和方法论的人来说，这种遭遇完全是不符合我对这个世界的认知的，因为按照概率论来说，我拿到好牌的概率应该是和别人一样的，我百思不得其解。于是这几天输了很多钱，没错，是一大把钱！！！

同学们解释为我运气不好，对于我来说，这个论断是没有任何实际根据的。因为我从来不认同“运气”和“命运”这种事情，我觉得任何事情都是有理由的，没有理由和原因的事情是根本不存在的。

然后我回去看《博弈论》，说到纳什均衡，纳什均衡指的是在非合作博弈里面，不管别人怎么选，自己都可以在某种程度上达到最优。然后我恍然大悟，我输钱的原因不在于概率的问题，而是在于我花在每个概率上加权的砝码不一样，也就是说局部非最优的场次特别多，导致全局最优的概率变小，输钱也在情理之中了。

所以加权的赢钱方式是：（收入-投入）对手拿到比你牌小的概率 最后加权赌钱的局数
而投入的合理公式应该是：投入= （收入/投入）你判断对手牌比你小的正确率*常数系数
收入的公式应该是：收入=自己的投入（心智，财富状况、心情、习惯）+他人的投入（心智、财富状况、心情、习惯）

在这里面，起决定作用的是，

1、你的心智
2、别人的心智
3、对手拿到牌比你小的概率
4、你判断对手牌比你小的概率

心智在这里怎么理解呢,可以理解为。控制自己，不要情绪化出牌，不要有所谓的不信邪的心理，不要在输的时候情绪化使劲上钱赌博。

该怎么做的时候就该怎么做，比如单A可以上的时候，不要因为输了钱就不上，跑了，这样只会导致越输越多。也不要因为输了钱就使劲想捞回本，

小点就刷涮的往上上，这也是错误的心态。坚持自己赌钱的原则，肯定就会赢。

然后就是，对手拿到牌比你小的概率，这个是没有办法控制的事情，当然，这个我们是可以依靠自己手里的牌计算出别人拿到比自己大的牌的概率的。

抓到金花的概率是多少，豹子的概率是多少，是可以计算出来的。具体的可以研究下概率论，就不深入讨论了。

因为和同学打着玩，所以也没必要这么认真的研究概率啦，挺累的。

看的头晕的可以记住以下几点简单的道理：

1、及时止损 比如说，人少的时候输的多就一开始就看牌。人多的时候就闷牌，因为人多的话，看牌早是会很吃亏的，闷牌的话投入产出比也很难说，如果经过专业训练，这个概率是可以计算出来的，因为人越少赚钱的概率就越大
2、越是那种情绪化严重的人，越会输钱。我观察过很多人，那种情绪化很严重的，控制情绪能力不强的人，一般就会输很多钱。我之前输钱就是情绪控制力上出了点问题和对概率论的理解偏差。

3、察言观色很重要，了解每个人的性格，就会增加你判断他牌大小的准确性。所以最好的赌钱表情是面瘫。当然，如果这个你学不会的话，也无所谓。及时止损和理性才是赌钱当中最关键的一点。还有，不要贪婪。
明白了这个道理，我输的钱都赢回来了。

其它具体的细节可以再自己研究嘛，真正的赌神，都是靠概率来赢钱的，不是靠运气和作弊。

confluence数据库校对规则的问题

2014/1/15

字符集和校对规则

字符集是一套字符和对应编码的集合，而对应的校对规则是对字符之间进行比较的一个或多个规则的集合。

举个简单的例子：

A——0

B——1

C——2

a——3

b——4

c——5

类似于上面的这样的，左边的集合属于字符集，右边的集合属于编码集。

而校对规则就是用于字符之间互相比较的一套规则：

比如我设定一个大小写敏感的规则，那么它就会把‘A’字符和‘a’字符当做两种不同的字符。

但如果我设定一个大小写不敏感的字符规则，那么它就会把‘A’字符和‘a’字符当做完全一样的字符来进行校对处理。

Mysql的校对规则和字符集设定

Mysql 5.1支持如下特性

使用多种字符集来存储字符串
使用多种校对规则来比较字符串
在同一台服务器、同一个数据库或甚至在同一个表中使用不同字符集或校对规则来混合字符串
允许定义任何级别的字符集和校对规则
由此我们知道，Mysql服务器、数据库、表单、列，这些的校对规则和字符集都是可以不一样的。

也就是说，你可以设定数据库的字符为utf8,校对规则utf8bin，表单的字符为utf8,校对规则为utf8general_ci，列再使用别的字符和校对规则，这是没问题的。

Confluence测试环境遇到的问题

上周在测试环境测试Confluence，我把我们线上环境里面的数据库导入到按照官网设置的空白数据库。

数据库的创建：

CREATE DATABASE confluence CHARACTER SET utf8 COLLATE utf8_bin;
数据库的导入

mysql – u root -p wiki < mysqldump.sql
启动confluence，无法启动，总是报编码校对的错误。显示报错 ‘Detected tables with non-default character encoding/collation’ Message

我按照官网提供的解决方法，修改了表单的字符校对规则，将所有80个表单的校对规则改成了utf8_bin.

类似于这样：

ALTER TABLE wiki.AO_187CCC_SIDEBAR_LINK CONVERT TO CHARACTER SET utf8_bin;
重新启动，结果发现还是同样的问题，编码校对的错误

然后经过同事的提示，去线上环境看了下，发现校对规则全部都是utf8generalci，于是改成了下面的格式

ALTER TABLE wiki.AO_187CCC_SIDEBAR_LINK CONVERT TO CHARACTER SET utf8generalci;
结果改完了还是这样的问题，最后彪哥解决了问题，重新创建了数据库，这样就可以了。

CREATE DATABASE wiki DEFAULT CHARACTER SET ‘utf8’;
原因就是，创建utf8字符集的数据库的时候，默认的校对规则是utf8generalci，我们的数据库最早创建的时候没有参考官方的这个教程，导致我们所有的表单也好，列也好，都是utf8generalci的校对规则。

然后我在写sql进行转换的时候，只转换了数据库和表的校对规则，更改列的校对规则的时候，报错了，当时也不知道是为什么。

总结一下：

解决这个问题的方法

1、改变新建数据库的默认编码（我们的办法）

2、转换整个数据库里面的数据的编码（官网的建议—但修改校对规则会有报错，当时没看懂，有空再验证一下）

念念不忘，必有回响—2013年总结

2014/1/5

1、读书

收获：

2013年读了68本书，平均五天一本：http://www.yuedudna.com/users/48599262/book/2013

视野得到了进一步的提升，见到的东西多了，看到的观点多了，自然思考就会有源头，不至于禁锢在一个狭小的范围内，所以我的收获就是

看到了更多的东西。

不足：

过于偏向于数量，却没有在阅读质量的提升上花费太多的时间。阅读笔记写的很少，总结也不够多。

新一年的展望：

希望能够建立自己的知识体系，以后阅读专注的方向：主要偏向于心理学，逻辑思维方面的书，计算机方面的数据库，Linux，需要深入一些。

算法和编程方面还是比较薄弱，需要进一步的加强和不断的练习。

2、交流

收获：

和组内的交流给了我很多书本上学不到的东西，譬如如何推动一件事情，如何坚持一些事情，如何去学习。

不足：

面对事情经常会有畏难情绪，对于比较dirty的活还是有些排斥，情绪控制的不太好，需要进一步的锤炼自己。和人交流的主动性仍然不够，只是在deadline的时候才会主动去寻求帮助。

需要改变。

新一年的展望：

控制自己的情绪，处理好杂乱的事情的同时也要多主动找人交流沟通，发现自己的不足。

3、思考

收获：

之前我每一天晚上都会花上一个小时的时间来思考自己遇到的问题，视野的提升给我的好处就是，思考问题稍微比较客观和从容了一些，愤青的时候更少了点。

不足：

发现虽然思考的不少，但是有时候情绪上来的时候容易钻牛角尖，听不去别人的建议，很多问题思考的太过理想化，以后要注意做事之前得多角度的思考不能太过偏激。

新一年的展望：

心态放平和一些，不能太过焦虑和急躁，否则容易走错路，做错事情。多听听别人的想法和建议，但是，坚持自己的原则，依然很重要。

4、成长

收获：

刚来的时候美团月销售额10亿，现在已经突破20亿了，时间却只有半年，但回头想想，很多事情都是在发生巨大变化的，不管是从团队，还是个人，还是我们在做的事情。

我在这大半年里，除了技能上的提高以外，思维方式和处事方式也有了很大的改变，当然，对于我自己来说，最大的莫过于自己视野的提升吧。

不足：

成长最慢的时期，恰好是自己情绪波动最大的时期，这个告诉我的一件事就是：“想的太多未必是一件好事。”当然，也不可能是坏事。缺点就是在能力不够的情况下，想法比较多，较浮躁。

新一年的展望：

之前上学的时候老师告诉我的一句话就是：“一个人有太多想法，和一个人没有想法，是没有区别的。”虽然我不太认同，但某种程度上还是很有道理的，多思考，谨慎下结论。

努力坚持，不是有一句话说嘛：“但行好事，莫问前程”。

一些感慨：

环境能造就人，是的没错。但我们作为一个人，我们和动物的差别就在于人能改变环境，就像我以前总是抱怨学校不好，工作刚开始的时候抱怨事情做的没挑战性一样，是弱者的表现。

做事情就像我读书买书看书，虽然很多人都不以为然，但我一直就这样做下去了，然后就潜移默化的会影响到很多身边的人开始读书，开始思考问题，我就觉得这是我的胜利。

有些时候我们为什么喜欢逃离，因为我们害怕。很喜欢《一代宗师》里面的一句话——“念念不忘，必有回响”

需要感谢的人有很多，就不一一说出来了。我不是王总的粉丝，但我挺喜欢他说的话：“对未来越有信心，对现在就越有耐心。”

用来与诸位共勉，顺祝新年快乐。
